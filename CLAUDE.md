# 專案目標  
建立一個靜態說明網站，作為教學方案說明，並適合部署於 GitHub Pages。
請依序完成下述步驟，並於開始前列出簡明的3-7項概念性檢查清單作為執行計畫：  
1. 分析網站結構與內容需求  
2. 設計統一模板與響應式排版  
3. 切分並製作對應 Markdown/Jekyll 檔案  
4. 處理資產、樣式與引用  
5. 驗證每頁屬性與無障礙規範  
6. 檢查編碼格式與語言標示  
7. 準備部署至 GitHub Pages
執行檔案產生與設計規劃後，請於每個步驟完成時檢查成果是否達到需求，如有不符請即時簡要修正並註明所做調整。
# 網站結構需求  
- 必須包含以下頁面：  
  1. **首頁（index）**：介紹整體背景、目標，以及橫跨七、八年級的融入式學習方案。  
  2. **七年級頁面**：說明基礎技能與數位公民素養養成階段，內容分為三個階段。  
  3. **八年級頁面**：說明專題應用與資訊思辨深化階段，內容分為兩個主題。  
  4. **行政與設備配套建議頁面**
- 每頁需具備：  
  - 頁面標題  
  - 主要內容區塊  
  - 返回首頁的導覽列  
  - 統一模板設計（如一致的 header、footer 與導覽列）
- 檔案結構建議：  
  - 每個主題頁請獨立一個檔案（如 `grade7.html`、`grade8.html`、`admin.html`），首頁為 `index.html`。  
  - 建議採用 Markdown（.md）或用 Jekyll 格式產生靜態 HTML 網頁，以確保符合 GitHub Pages 的格式需求。  
  - 所有網站資產（如圖片、CSS）請放在 `assets` 資料夾，並於需要處正確引用。
# 設計注意事項  
- 網站必須具備基本的響應式設計，支援桌上型與行動裝置瀏覽。  
- 每頁 HTML 請設定 UTF-8 編碼及語言標註 `lang="zh-Hant"`。  
- 插入圖片時，請加入適當的 `alt` 屬性，以提升無障礙設計與通用瀏覽。  
- 若有缺失頁面或資源，需顯示友善錯誤訊息，例如「找不到頁面或資源，請回到首頁」。
# 目錄結構範例  
```  
/  
├── index.md            # 首頁  
├── grade7.md           # 七年級頁面  
├── grade8.md           # 八年級頁面  
├── admin.md            # 行政與設備配套建議頁面  
├── _config.yml         # Jekyll 設定檔（如需要）  
└── assets/  
    ├── images/         # 所有圖片放置於此資料夾  
    └── styles.css      # 網站共用樣式表  
```
# 輸出指引  
- 所有網頁應具備統一的樣板（header、footer、導覽列）。  
- 每個頁面必須有標題、說明內容，並附帶返回首頁與各頁之快速導覽。  
- 圖片需添加 alt 屬性。  
- 全站採用 UTF-8 編碼與 `lang="zh-Hant"`。  
- 若有缺失資源，顯示「找不到頁面或資源，請回到首頁」。
