## 無障礙設計檢查

### 所有頁面必須符合以下規範：

#### 1. 編碼與語言
- [x] 所有 HTML 檔案使用 UTF-8 編碼
  ```html
  <meta charset="UTF-8">
  ```
- [x] 所有頁面設定語言標籤為繁體中文
  ```html
  <html lang="zh-Hant">
  ```

#### 2. 語意化結構
- [x] 使用正確的標題階層（h1 → h2 → h3）
- [x] 使用語意化標籤（header、nav、main、section、footer）
- [x] 列表使用 `<ul>` 或 `<ol>` 標籤

#### 3. 連結與互動元素
- [x] 所有連結具備焦點樣式（`:focus`）
- [x] 返回頂部按鈕包含 `aria-label` 屬性
  ```html
  <a href="#" class="back-to-top" aria-label="返回頁面頂部">↑</a>
  ```
- [x] 連結文字具描述性（避免「點此」等模糊字眼）

#### 4. 圖片（如有使用）
- [ ] 所有圖片包含 `alt` 屬性
  ```html
  <img src="assets/images/example.jpg" alt="具體描述圖片內容">
  ```
- [ ] 裝飾性圖片使用空白 alt：`alt=""`

#### 5. 顏色對比
- [x] 文字與背景對比度符合 WCAG 2.1 AA 標準
- [x] 連結顏色與周圍文字有足夠對比

## 快速部署指令（Git 使用者）

```bash
# 確認所有檔案已加入
git status

# 加入所有新檔案
git add .

# 建立 commit
git commit -m "Update: [描述你的修改]"

# 推送到 GitHub
git push origin main
```

**注意事項：**
1. 確保所有檔案使用 UTF-8 編碼儲存
2. Windows 使用者注意換行符號（建議使用 LF 而非 CRLF）
3. 上傳前建議先在本地瀏覽器測試所有頁面
4. GitHub Pages 部署需要 1-2 分鐘，請耐心等候
